Restauration

 

 

Le dialogue de restauration présente une interface quelque peu simplifiée vers l'outil pg_restore de PostgreSQL.

Après avoir sélectionné un fichier de sauvegarde valide, pgAdmin lira le contenu du fichier une fois que vous aurez appuyé sur le bouton Visualiser. Cela peut prendre du temps, suivant la taille du fichier. Vous pouvez suivre la progression sur l'onglet Messages. Après que l'examen se soit terminé, pgAdmin affichera le contenu dans l'onglet Contenu.

L'outil ne créera pas la base de données à restaurer. Pour restaurer une base de données, vous devez tout d'abord la créer en utilisant pgAdmin, puis démarrer l'outil de restauration avec la nouvelle base de données sélectionnée. Les données seront restaurées dans cette base.

Vous pouvez ne restaurer qu'un seul objet quand vous être dans le mode de restauration base (c'est affiché dans le titre de la fenêtre, voir l'exemple). Après avoir examiner le contenu du fichier de sauvegarde, vous pouvez sélectionner la table désirée à partir de l'onglet Contenu et cliquer sur le cas Objet seul. Seule cette table sera restaurée.

Vous pouvez sélectionner une table comme destination de restauration. pgAdmin restaurera les données si elles sont présentes dans le fichier de configuration.

Pour utiliser la sauvegarde, l'outil pg_restore doit être accessible par pgAdmin. Cela peut se faire en le rendant disponible dans le PATH système ou en le copiant dans le même répertoire que celui de l'exécutable pgadmin3.
