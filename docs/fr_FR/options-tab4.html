Options de pgAdmin - Onglet 4 (Requêtes)

  • Police - Cette option vous permet de sélectionner la police utilisée dans les champs texte de saisir des requêtes SQL. Ceci peut être utile pour les pays avec des jeux de caractères multi-octets dont l'affichage est mal gérée par les polices standards.
     
  • Nombre de lignes maximum à récupérer - Cette option spécifie le nombre de lignes de statistiques sur les jobs et les étapes des jobs lors de la visualiation des statistiques dans le navigateur principale. Contrairement aux statistiques pour les autres objets, statistiques qui consistent en un nombre de lignes fixes, une ligne est créée à chaque fois qu'un job ou une étape est exécutée. Les statistiques les plus récentes seront affichées.
     
  • Nombre maximum de caractères par colonne - Cette option spécifie la longueur maximale des données affichées dans une seule colonne.
     
  • Compter les lignes si nombre estimé est inférieur à - Dans le navigateur, si le nombre estimé de lignes dans une table (lu à partir des statistiques de la table) est sous cette limite, un SELECT count(*) sera exécuté pour trouver la valeu exacte.
     
  • Indenter les caractères - Cette option spécifie le nombre de caractères pour faire l'indentation dans les champs SQL.
     
  • Utiliser des espaces à la place des tabulations - Si coché, quand vous utilisez la touche tabulation dans un champ texte SQL, des espaces seront insérés à la place de la tabulation.
     
  • Guillemets pour la copie des résultats - Cette option spécifie comment les champs doivent être mis entre guillemets quand les données sont copiées dans le presse-papier à partir de la liste des résultats. La mise entre guillemets peut seulement être appliquée aux colonnes de type chaîne de caractères (c'est-à-dire que les colonnes numériques n'auront pas de guillemets) ou toutes les colonnes quelque soit le type de données. Désactiver cette option pourrait empêcher le copier/coller des lignes de la grille d'édition si des caractères comme les retours à la ligne sont impliqués.
     
  • Caractère guillemet pour la copie des résultats - Cette option spécifie le caractère utilisé pour mettre entre guillemets les valeurs copiées de la liste de résultats dans le presse-papier. Elle n'a pas d'effet si Guillemets pour la copie des résultats est configuré à rien.
     
  • Séparateur de champ pour la copie des résultats - Cette option spécifie quel caractère doit être utilisé pour séparer les champs copiés de la liste de résultat dans le presse-papier.
     
  • Utiliser la caractère TAB pour la complétion automatique - Par défaut, la complétion automatique dans l'outil de requêtage est activée en utilisant la combinaison de touches Controle+Espace. Cette option vous permet d'utiliser aussi la touche Tabulation. Néanmoins, il signifie aussi que vous ne pouvez pas saisir le caractère Tab.
     
  • Copier SQL de la fenêtre principale au dialogue SQL - Lors de l'ouverture de l'outil de requêtage, avec cette option activée, la définition de l'objet sélectionné dans le navigateur sera insérée en tant que requête par défaut de l'éditeur.
     
  • Afficher les valeurs NULL en tant que <NULL> ? - Cette option fera que les valeurs NULL seront affichées <NULL> dans la grille des résultats de l'outil de requêtage. Notez que cette option n'affecte pas les données qui sont exportées à partir de la grille, cela n'affecte que l'affichage.
